#### 2. Mobile Navigation JavaScript
**File Created:**
- `app/assets/js/mobile-nav.js` (New file, 80+ lines)

**Implementation Details:**

##### Self-Executing Function (IIFE):
```javascript
(function() {
  'use strict';
  
  function initMobileNav() {
    // Create hamburger button dynamically
    const hamburger = document.createElement('button');
    hamburger.className = 'hamburger-menu';
    hamburger.setAttribute('aria-label', 'Toggle navigation menu');
    hamburger.innerHTML = `
      <div class="hamburger-icon">
        <span></span>
        <span></span>
        <span></span>
      </div>
    `;
    
    // Create overlay
    const overlay = document.createElement('div');
    overlay.className = 'sidebar-overlay';
    
    // Insert at start of body
    document.body.insertBefore(hamburger, document.body.firstChild);
    document.body.insertBefore(overlay, document.body.firstChild);
    
    // ... (rest of implementation)
  }
  
  // Initialize when DOM is ready
  if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', initMobileNav);
  } else {
    initMobileNav();
  }
})();
```

**Toggle Function:**
```javascript
function toggleSidebar() {
  hamburger.classList.toggle('active');
  sidebar.classList.toggle('active');
  overlay.classList.toggle('active');
  
  // Prevent body scroll when menu is open
  if (sidebar.classList.contains('active')) {
    document.body.style.overflow = 'hidden';
  } else {
    document.body.style.overflow = '';
  }
}
```

**Event Listeners:**
```javascript
// Click hamburger to toggle
hamburger.addEventListener('click', toggleSidebar);

// Click overlay to close
overlay.addEventListener('click', toggleSidebar);

// Click nav link to close (mobile)
if (window.innerWidth <= 768) {
  const navLinks = sidebar.querySelectorAll('.nav-link');
  navLinks.forEach(link => {
    link.addEventListener('click', () => {
      if (sidebar.classList.contains('active')) {
        toggleSidebar();
      }
    });
  });
}

// Close on Escape key
document.addEventListener('keydown', (e) => {
  if (e.key === 'Escape' && sidebar.classList.contains('active')) {
    toggleSidebar();
  }
});
```

**Window Resize Handler:**
```javascript
let resizeTimer;
window.addEventListener('resize', () => {
  clearTimeout(resizeTimer);
  resizeTimer = setTimeout(() => {
    if (window.innerWidth > 768) {
      // Desktop view - reset everything
      hamburger.classList.remove('active');
      sidebar.classList.remove('active');
      overlay.classList.remove('active');
      document.body.style.overflow = '';
    }
  }, 250); // Debounce for performance
});
```

## Mobile UX Flow

### Desktop (> 768px):
1. Sidebar is permanently visible on the left
2. Hamburger button is hidden
3. Main content has left margin to accommodate sidebar
4. No overlay needed

### Mobile (<= 768px):
1. **Initial State:**
   - Sidebar is hidden off-screen (left: -280px)
   - Hamburger button is visible (top-left corner)
   - Main content takes full width

2. **User Taps Hamburger:**
   - Hamburger animates to "X" shape
   - Sidebar slides in from left (left: 0)
   - Dark overlay appears over content
   - Body scroll is disabled

3. **User Taps Overlay / Nav Link / Esc:**
   - Hamburger animates back to "â˜°" shape
   - Sidebar slides out to left (left: -280px)
   - Overlay fades out
   - Body scroll is re-enabled

## Technical Decisions

### 1. Why IIFE for mobile-nav.js?
```javascript
(function() { /* code */ })();
```
**Rationale:** Prevents global namespace pollution. All variables and functions are scoped locally, avoiding conflicts with other scripts.

### 2. Why Dynamic DOM Creation?
```javascript
const hamburger = document.createElement('button');
document.body.insertBefore(hamburger, document.body.firstChild);
```
**Rationale:** 
- No need to modify HTML in 5 different files
- Ensures hamburger is always first element (z-index top priority)
- Easier to maintain - changes only needed in one JS file

### 3. Why Body Overflow Lock?
```javascript
document.body.style.overflow = 'hidden'; // When menu open
```
**Rationale:** Prevents background scrolling on mobile when sidebar is open, improving UX and preventing visual glitches.

### 4. Why Debounce Resize Handler?
```javascript
clearTimeout(resizeTimer);
resizeTimer = setTimeout(() => { /* ... */ }, 250);
```
**Rationale:** Resize events fire rapidly (100+ times during a drag). Debouncing ensures cleanup logic runs only once after resizing stops, improving performance.
